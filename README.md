# Hormuz Strait Tracker

Трекер суден, що проходять через Ормузьку протоку, на основі публічних AIS-даних (AISstream.io).

## Як це працює

1. `collector.js` — постійно (24/7) підключений по WebSocket до AISstream, слухає всі AIS-повідомлення
   в bounding box навколо протоки, і для кожного судна пам'ятає його попередню позицію.
   Коли судно перетинає умовний меридіан (56.25°E, налаштовується в `config.js`),
   подія дописується у `data/crossings.jsonl`.
2. `server.js` — маленький Express-сервер, що читає накопичені події, агрегує їх
   (скільки і яких типів пройшло в кожну сторону) і віддає в `public/index.html`.

Це два окремі процеси — збирач має працювати постійно у фоні, незалежно від того,
чи відкрив хтось сайт у браузері.

## Запуск

```bash
npm install
cp .env.example .env
# встав свій ключ у .env — отримати безкоштовно на https://aisstream.io/authenticate
```

У двох окремих терміналах:

```bash
npm run collect   # запускає збирач даних (тримати запущеним постійно)
npm start         # запускає сайт на http://localhost:3000
```

## Куди рухатись далі

- **Деплой збирача.** Локально скрипт зупиниться, коли закриєш термінал/ноутбук.
  Для постійної роботи задеплой `collector.js` на Render / Railway / fly.io (є безкоштовні тіри
  для легких фонових воркерів) або підніми на VPS з `pm2`/`systemd`.
- **База даних замість JSONL.** Для пет-проекту `crossings.jsonl` цілком достатньо, але якщо
  зʼявиться потреба у складніших вибірках (за датою, судном, портом призначення) —
  легко перейти на SQLite, змінивши лише `storage.js`.
- **Калібрування лінії/гістерезису.** Значення в `config.js` — orієнтовні. Подивись живі
  треки суден у протоці (наприклад, через безкоштовний перегляд на MarineTraffic чи VesselFinder,
  не через API) і скоригуй `CROSSING_LONGITUDE` та `CROSSING_LAT_RANGE` під реальну географію
  фарватеру, щоб не ловити судна, що просто йдуть паралельно березі.
- **Легкі помилки типу судна.** `ShipStaticData` приходить рідше за позиції — деякі судна
  якийсь час матимуть `category: "Unknown"`, поки не прийде перше статичне повідомлення.
  Це нормально, з часом дані доповнюються.

## Структура

```
config.js          # координати протоки, лінія перетину, параметри
vesselTypes.js      # мапінг AIS-кодів типів суден у категорії
storage.js          # запис/читання стану суден і подій перетину
collector.js         # WebSocket-воркер: слухає AIS, детектує перетини
server.js            # Express-сервер: віддає статистику й фронтенд
public/index.html    # дашборд
data/                # тут накопичуються crossings.jsonl і vessel-state.json
```
